Есть утилитный класс с методами по работе с sql.
в SQL есть PreparedStatement, когда параметры пишутся через '?', а потом значения передаются отдельно (массивом).
Получился такой баг:
есть sql вида:

select * from table_X where
col1=?
and array_col=? or array_col=?
and col2 = ?
and col3 in (?,?,?)

Получается, что jdbc методы массив[] оставляют как один ?
а Collection разворачивают в ?,?,?

Вот как выглядят параметры, которые подаются на вход в метод

Индекс 0: Строка со значением "value"

Индекс 1: Массив строк ["tag1", "tag2"]

Индекс 2: Массив строк ["tag3", "tag4"]

Индекс 3: Строка со значением "%pattern%"

Индекс 4: Объект множества (ImmutableCollections$SetN) размером 3 элемента:

"value2"

"value1"

"value3"
